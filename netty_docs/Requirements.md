# Netty 프로젝트 요구사항

## 1. 프로젝트 개요
**Netty**는 무심코 적은 "낙서(메모)"를 구조화된 "자산(회고/인사이트)"으로 변환해주는 성장 관리 서비스입니다. 기록을 꾸준히 남기는 것에 어려움을 겪는 사용자들에게 기록의 심리적 장벽을 낮추고, 성장을 시각적으로 보여주는 것을 목표로 합니다.

### 가치 제안 (Value Proposition)
- **Scribble to Asset (기록의 자산화)**: AI가 단순한 메모를 구조화된 기록으로 변환합니다.
- **Alternative Signal (대체 불가능한 신호)**: 꾸준함과 메타인지를 시각화하여 새로운 형태의 "스펙"으로 증명합니다.
- **Community as a Pacer (러닝메이트)**: 함께 성장하는 환경을 제공합니다.

## 2. 기술 스택 및 제약사항
### 클라이언트 (`netty_client`)
- **프레임워크**: React + Vite
- **스타일링**: 
    - **TailwindCSS 사용 안 함**.
    - **CSS Modules 사용 안 함**.
    - **BEM 네이밍 컨벤션** (Block Element Modifier) 준수.
    - 각 JSX 컴포넌트마다 대응되는 별도의 CSS 파일 생성 (예: `Component.jsx` + `Component.css`).
- **배포**: AWS + Nginx (추후 예정).

### 서버 (`netty_server`)
- **런타임**: Node.js
- **프레임워크**: Express.js
- **데이터베이스**: MongoDB (Atlas 옵션).
- **인증**: Passport.js (예정).
- **AI 통합**: OpenAI API (GPT-4o-mini).

## 3. 핵심 기능 요구사항

### A. 기록 시스템 (The Engine)
- **AI 초안 생성기**: 
    - 입력: 키워드 또는 짧은 문장.
    - 출력: 5단계 구조화된 텍스트 (계기 -> 상황 -> 생각 -> 감정 -> 보완).
- **역질문 봇 (Prompting)**:
    - 입력 내용이 빈약할 경우 AI가 구체화를 위한 후속 질문을 던짐.
- **성장 캘린더**:
    - 기록 밀도를 시각화한 GitHub 스타일의 잔디 심기 UI.

### B. 신호 체계 (The Signal)
- **성장도감 (Leveling)**:
    - 기록의 질(AI) + 지속성 + 동료 인증을 합산하여 레벨 산정.
- **역량 포트폴리오**:
    - 강점 키워드(예: #실행력, #위기관리)를 워드클라우드나 차트로 시각화.

### C. 커뮤니티 (The Support)
- **스쿼드 (Squads)**:
    - 유사한 목표를 가진 소규모 그룹.
- **목격 (Witnessing)**:
    - 무거운 댓글 대신 가벼운 반응(봤어요/멋져요/궁금해요) 위주.

### D. 관리자 (Admin)
- **CRM 대시보드**:
    - 사용자 이탈 및 성장 정체 모니터링.
    - 자동/수동 개입 트리거.

## 4. 타겟 고객
- **초기 수용자 (Early Adopters)**: 학벌/경력보다 '실력'과 '태도'로 증명하고 싶은 "실행가".
- **성장 고관여층**: 성장에 대한 욕구는 높으나 구체적인 방법론이 없는 사람들.
