### 1. 일상 기록, 성장 기록을 하나로 합치기
탭 아래 [상세 탭]으로 구분
현재 일상 기록(daily-record), 성장 기록(growth)는 상세 탭을 누르면 그대로 나오게

### 2. 성장 기록(GrowthRecordPage) 대폭 수정
#### 2.1. 메뉴 레이아웃
1. 상세 메뉴로 [성장 계획], [월 성장지표], [월 성장분석], [xx년 한눈에 정리], [연 결과보고서], [성장소감] 으로 구성.
#### 2.2. 성장 계획
다음과 같은 사항이 들어가야 함
1. 원하는 나: (가치/방향)
2. 성장목표: (하고 싶은 일)
3. 성장활동: (필요한 일) 여러개
4. 성장동기: (하고 싶은 일들)

{1,2,3,4} 가 합쳐져(1:1:N:1) 하나의 객체가 되고, 이 객체는 동적으로 생성 가능.
즉, (1,2,3,4)를 합친 객체를 동적으로 생성 / 삭제할 수 있어야 하고, 객체 내 [3] 요소를 동적으로 생성/삭제할 수 있어야 함.

초기 객체의 수는 3개, 객체 내 [3] 요소의 수도 3개.

### 2.3. 월 성장지표
[2.2. 성장 계획]은 연 1회 작성되며, [월 성장지표]는 월 1회 작성이 원칙. (또는 2주 1회 등으로 바뀔 수 있으나, 연간 여러 번 작성됨)

2.2.1(원하는 나), 2.2.2(성장 목표), 2.2.3(성장활동)을 그대로 가져오고(1:1:N), 각 성장 활동(N1 + N2 + ... )에 따라 활동일지(세부 실천방안 및 소감) 작성

#### 2.4. 월 성장분석
[월 성장분석]은 [월 성장지표]와 마찬가지로 월 1회 작성이 원칙. (또는 2주 1회 등으로 바뀔 수 있으나, 연간 여러 번 작성됨)

2.2.1(원하는 나), 2.2.2(성장 목표)를 그대로 가져오고 (1:1), 각 [원하는 나]에 따라 행동결과(강점, 약점분석), 행동보완(성장 환경조성)을 작성

### 2.5. 20xx년 한 눈에 정리
연말에 작성. 12개의 월별 란이 있고, 각 [란]은 활동내용, 핵심요약으로 구성

#### 2.6. 20xx년 결과보고서
연말에 1회 작성.

2.2. [성장 계획] 에서 2.2.1(원하는 나), 2.2.2(성장 목표), 2.2.3(성장활동)을 그대로 가져오고(1:1:N), 각 성장 활동(N1 + N2 + ... )에 따라 성장 활동성과(이뤄낸 일)을 작성

#### 2.7. 나의 성장소감
연말에 1회 작성. 다음과 같이 구성
- 20xx년 한 문장 요약
- 본문
